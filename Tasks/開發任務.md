## 🚀 並行開發策略（重要）

> **⚠️ 本專案支援多 Agent 並行開發，可節省約 56% 開發時間**

### 並行開發層級

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Layer 1: Phase 內部並行（同一 Phase 內的獨立任務）                        │
│  Layer 2: Phase 之間並行（Phase 4-16 頁面可全部並行）                      │
│  Layer 3: AI 工具並行（Claude 結構 + Gemini 文案同時進行）                 │
└─────────────────────────────────────────────────────────────────────────┘
```

### 任務依賴關係圖

```
Phase 1-3 (串行) → Phase 4-16 (並行 13 頁面) → Phase 17-19 (並行 3 優化) → Phase 20
     ↓                    ↓                           ↓
  基礎設施             各頁面獨立                  優化互不依賴
```

### 並行標記說明

| 標記 | 含義 |
|------|------|
| `🔀 PARALLEL` | 此區塊內的任務可並行執行 |
| `⏳ SEQUENTIAL` | 此區塊必須串行執行 |
| `🔗 DEPENDENCY` | 依賴前置任務完成 |

### Sub-Agent 啟動指令

#### 啟動 Gemini CLI（文案/翻譯任務）
```bash
gemini -m gemini-2.5-pro -y "YOUR_PROMPT_HERE"
```

#### 啟動 code-reviewer Sub-agent（每個 Phase 完成後必須執行）
```
[指令] 使用 code-reviewer Sub-agent 審查 Phase N 的所有代碼
```

#### 啟動 codex-cli Skill（演算法審查）
```
[指令] 使用 codex-cli Skill 審查/優化演算法代碼
```

#### 啟動 debugger Sub-agent（遇到 Bug 時）
```

```

## 核心原則

### 1. 共用模組原則（最重要）

> **所有可共用的功能，只製作一個共用模組，避免修改時要修改所有頁面**

#### 必須共用的模組

| 優先級 | 模組 | 檔案 | 使用頁面 |
|--------|------|------|----------|
| P0 | 導航列 | `js/components/navbar.js` | 所有頁面 |
| P0 | 側邊欄 | `js/components/sidebar.js` | 所有內頁 (12頁) |
| P0 | 頁腳 | `js/components/footer.js` | Landing Page |
| P0 | 主題切換 | `js/theme.js` | 所有頁面 |
| P0 | 語言切換 | `js/i18n.js` | 所有頁面 |
| P1 | Toast 通知 | `js/components/toast.js` | 需要通知的頁面 |
| P1 | Modal 對話框 | `js/components/modal.js` | 需要對話框的頁面 |
| P1 | 頁面模板 | `js/components/page-template.js` | 所有內頁 |

#### 開發順序原則

```
1. 先開發共用模組 → 2. 再開發頁面內容
```

**禁止做法**：
- ❌ 在每個頁面複製貼上導航/側邊欄 HTML
- ❌ 每個頁面各自維護導航/側邊欄狀態

**正確做法**：
- ✅ 共用模組以 JavaScript 動態渲染
- ✅ 頁面只包含內容區域，共用部分由 main.js 載入

---

### 2. AI 工作分配規則（強制執行）

> **⚠️ 嚴格執行：每個任務必須按照指定的 AI 工具執行，不可省略**

#### 2.1 AI CLI 分工

| AI | 標籤 | 擅長任務 | 調用方式 |
|----|------|----------|----------|
| **Claude Code** | `[CLAUDE]` | 主要開發、HTML/CSS/JS 編寫 | 直接執行 |
| **Gemini CLI** | `[GEMINI]` | 文案生成、翻譯、長文本、研究 | `gemini -m gemini-2.5-pro -y "prompt"` |
| **Codex CLI** | `[CODEX]` | Code Review、演算法、效能優化 | `codex-cli` Skill |

#### 2.2 Sub-agents

| Agent | 標籤 | 用途 | 觸發時機 |
|-------|------|------|----------|
| `code-reviewer` | `[REVIEW]` | 代碼品質審查 | **每個 Phase 完成後必須執行** |
| `debugger` | `[DEBUG]` | 問題排查 | 遇到 Bug 或異常行為時 |
| `doc-generator` | `[DOC]` | 文檔生成 | 需要生成/更新文檔時 |
| `researcher` | `[RESEARCH]` | 快速研究 | 需要調查最佳實踐時 |

#### 2.3 Skills

| Skill | 標籤 | 用途 | 觸發時機 |
|-------|------|------|----------|
| `rwd-design` | `[RWD]` | 響應式設計指南 | 響應式相關任務 |
| `codex-cli` | `[CODEX]` | Codex CLI 調用 | 演算法/審查任務 |
| `gemini-cli` | `[GEMINI]` | Gemini CLI 調用 | 文案/翻譯任務 |

#### 2.4 強制執行規則

```
┌─────────────────────────────────────────────────────────────────┐
│  【強制】Phase 完成檢查點                                        │
│                                                                 │
│  每完成一個 Phase，必須執行：                                    │
│  1. 使用 code-reviewer Sub-agent 審查該 Phase 所有代碼          │
│  2. 修復審查發現的問題                                          │
│  3. 確認無問題後才能進入下一個 Phase                             │
└─────────────────────────────────────────────────────────────────┘
```

**禁止行為**：
- ❌ 跳過任何標籤指定的 AI 工具
- ❌ 不使用 `[GEMINI]` 就自己寫翻譯/文案
- ❌ 不使用 `[CODEX]` 就自己審查複雜演算法
- ❌ 連續完成多個 Phase 不進行代碼審查
