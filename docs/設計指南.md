# NexusAI Glassmorphism 2026 - 設計指南

## 設計哲學

> "Faster, Smarter, More Inclusive, and More Human"

### 三大核心原則
1. **Human Layer** - 人性化設計，拒絕冰冷感
2. **Strategic Minimalism** - 戰略性極簡，每個元素必須有存在理由
3. **Purposeful Motion** - 有目的性的動態，服務清晰度非炫技

## Glassmorphism 設計原則

### 核心特徵
| 特徵 | 說明 | 建議值 |
|------|------|--------|
| 背景模糊 | backdrop-filter blur | 10-30px |
| 半透明 | rgba 透明度 | 10-30% |
| 邊框 | 半透明邊框增加深度 | 1px |
| 陰影 | 柔和陰影增加層次 | shadow-md |

### CSS 實作範例
```css
.glass-element {
  background: rgba(255, 255, 255, 0.25);
  backdrop-filter: blur(16px);
  -webkit-backdrop-filter: blur(16px);
  border: 1px solid rgba(255, 255, 255, 0.4);
  border-radius: 16px;
  box-shadow: 0 4px 6px rgba(0, 0, 0, 0.07);
}
```

### 最佳實踐
1. **確保對比度** - 文字對比度 ≥ 4.5:1
2. **適度使用** - 不要過度使用，保持清晰層級
3. **背景處理** - 複雜背景用更高模糊度
4. **Safari 兼容** - 記得加 `-webkit-backdrop-filter`
5. **回退方案** - 不支援時提供實色背景

### 何時使用
- 建立視覺層級和深度
- 卡片、導航、模態框
- 浮動面板

### 何時避免
- 過度使用（會造成視覺疲勞）
- 複雜背景且未充分模糊
- 低對比度環境

### 緩動函數
```css
/* 標準 */

--ease-out: cubic-bezier(0, 0, 0.2, 1);
```

### 動畫類型
| 類型 | 用途 | 時長 |
|------|------|------|
| 懸停 | 按鈕、卡片 | 150-300ms |
| 進入 | 頁面元素出現 | 300-500ms |
| 退出 | 元素消失 | 150-200ms |
| 滾動觸發 | 滾動到視口 | 500-700ms |

### 減少動畫偏好
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```
